What it does

Everything you need for a production torrent search service - without the boilerplate.

Multi-Source Search

7 indexers searched in parallel goroutines - PirateBay, 1337x, YTS, NyaaSI, LimeTorrents, TorrentProject, HiddenBay. Results merged in a single response.

Real-Debrid Integration

Full round-trip: add magnet → poll status → select files → unrestrict link. Encrypted key storage (AES-256-GCM). Background refresh keeps links fresh.

MongoDB Persistence

Sessions, users, favorites, stream URL cache, cover images, and generic KV store - all in MongoDB. Indexes created automatically on startup.

Background Jobs

Ticker-based scheduler for stream URL refresh (24 h), image cache pre-warming (6 h), storage cleanup (1 h), and log rotation (24 h). All observable from the dashboard.

Single Binary

go build -o server . produces one self-contained binary. No runtime, no package manager, no interpreter. Cold start under 100 ms.

Monitoring Dashboard

Browser-based dashboard at / with job status, API usage charts, database health, log viewer, and manual job triggers. Password + IP protected.

Fan-out Search

Every search spawns one goroutine per indexer. Results land on a buffered channel. No slow indexer blocks the response - all within the request context's deadline.

Storage Interface

All handlers receive a pkg/storage.Database interface, not a concrete driver. The MongoDB client is the active implementation; the interface makes the driver swappable.

Graceful Shutdown

SIGINT/SIGTERM triggers a 30-second drain: job scheduler stops first, then server.Shutdown(ctx) drains in-flight HTTP connections.

No Framework

The router is a thin wrapper around Go 1.22's standard net/http ServeMux with Express-style :param translation. Zero external router dependency.

Quickstart

Up and running in three steps.

1

Clone and build

git clone https://github.com/akshatsinghkaushik/torrent-search-go.git
cd torrent-search-go
go mod download
go build -o server .
2

Configure environment

cp .env.example .env
# Edit .env - minimum required:
# MONGODB_URI=mongodb://localhost:27017
# SESSION_SECRET=<64-char random hex>
# GOOGLE_SERVICE_ACCOUNT_JSON={...}
# GOOGLE_CUSTOM_SEARCH_ENGINE_ID=your-cse-id
3

Run

./server
# Server starts at http://localhost:3001
# Dashboard at http://localhost:3001/
# Health at http://localhost:3001/health

Docker alternative 

docker build -f deployments/Dockerfile -t torrent-search-go .
docker run -p 3001:3001 --env-file .env torrent-search-go

docker compose -f deployments/docker-compose.yml up

Configuration

Pure environment variables - no config files, no reinvention. Sensible defaults for everything optional.

MONGODB_URI
MongoDB connection string. Supports mongodb:// and mongodb+srv://. Credentials can be injected separately via MONGO_USERNAME / MONGO_PASSWORD.
SESSION_SECRET
Signs session tokens. Also used as fallback AES-256-GCM key for Real-Debrid key encryption if REAL_DEBRID_ENCRYPTION_KEY is not set.
GOOGLE_SERVICE_ACCOUNT_JSON
Full service-account JSON with OAuth credentials embedded. The server extracts oauth_client_id and oauth_client_secret automatically.
HIDDENBAY_URL
Override URL for the HiddenBay/PirateBay scraper. Useful if the default domain is blocked.
MONITORING_IP_ALLOWLIST
Comma-separated IPs/CIDRs allowed to access /api/monitoring/*. Combine with DASHBOARD_PASSWORD for two-factor protection.
ALLOWED_EMAILS
Comma-separated Google account emails permitted to authenticate. Leave empty to allow any Google account.

Frequently Asked Questions

What does torrent-search-go do?
It is an HTTP API backend written in Go that searches multiple torrent indexers simultaneously, caches results, integrates with Real-Debrid for premium streaming links, and provides Google OAuth user accounts with favorites, cover images, and background maintenance jobs.
Which torrent indexers are supported?
Seven scrapers are registered: PirateBay/HiddenBay, 1337x (via the 1337xx.to mirror, direct HTTP), YTS, NyaaSI, LimeTorrents, and TorrentProject. All are searched concurrently via goroutine fan-out.
What database does it use?
MongoDB. Indexes are created automatically on startup. Any MongoDB deployment works - local, Atlas free tier, or self-hosted. The storage layer is abstracted behind an interface making the driver swappable.
Is Real-Debrid required?
No. Torrent search, browsing, and details work without Real-Debrid. Stream URL resolution and the proxy endpoint require a user-provided Real-Debrid API key stored encrypted per-user in MongoDB.
Can I deploy it on Railway, Fly, Render, or Cloud Run?
Yes. The binary listens on $PORT (injected by most platforms) and defaults to 8080 in Docker. A health check at GET /health is ready for platform probes. railway.json and railpack.toml are included for Railway specifically.
